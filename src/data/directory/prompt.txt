HOW TO EXTRACT AND UPDATE LGU DATA
================================================================================

Step 1: Extract Data from Source
---------------------------------
1. Open the Google Sheets from DILG
2. Copy the relevant data for a specific region
3. Create a CSV file with the following format:
   - Column 1: locality_type (either "City" or "Municipality")
   - Column 2: LGU name
   - Column 3: position (either "mayor" or "vice_mayor")
   - Column 4: official's full name

Example CSV format:
```
City,Angeles City,mayor,CARMELO B. LAZATIN II
City,Angeles City,vice_mayor,AMOS B. RIVERA
Municipality,Apalit,mayor,OSCAR D. TETANGCO JR.
Municipality,Apalit,vice_mayor,PEDRO C. NUCOM
```

Step 2: Save the CSV File
--------------------------
Save the CSV file with a descriptive name like:
- region1-lgu.csv
- region2-lgu.csv
- region3-lgu.csv
- ncr-lgu.csv

Step 3: Run the Update Script
------------------------------
Use the update-lgu-from-csv.cjs script to update the main lgu.json file:

```
node scripts/update-lgu-from-csv.cjs region1-lgu.csv
```

Output Files:
-------------
- lgu.json: Updated main database file
- lgu-backup-[timestamp].json: Backup of original data
- lgu-not-found.json: List of LGUs that need to be added (Needs manual checking)

Handling Not Found LGUs:
------------------------
If the script reports LGUs as "not found":
1. Check lgu-not-found.json for the list
2. These LGUs need to be manually added to the correct province
3. Common issues:
   - Name variations (e.g., "Santa Maria" vs "Sta. Maria")
   - Abbreviations (e.g., "CSJDM" for "City of San Jose del Monte")
   - Missing cities or municipalities in the database
